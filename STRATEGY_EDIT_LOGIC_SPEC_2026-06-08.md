# 📜 사장님 사상 — 「전략 수정 모드」 단계별 진입가 + SL 정책 (2026-06-08)

> **사장님 명시 (2026-06-08)**:
> > "단계별 진입전략에서 자본 그리고 단계추가와 금액 트리% 등을 반복적으로 수정하면
> >  진행되지 않은 전략의 포지션 진입가가 변경되고 모든 단계별 포지션이 모두 실행되면
> >  투자금의 -80%가 되면 강제청산을 하게 되는데 이런것에 대한 로직을
> >  다시 한번더 기획해줘 수정을 할려고 들어오면 단계별 진입가를 알수 없어
> >  신중하게 다시 검정과 로직을 기획해서 개발을 할수 있게 해줘"

이 문서 = 영구 보존. 향후 모든 「수정 모드」 코드 변경 = 이 spec 100% 적용.

---

## 🌟 사장님 사상 핵심 (절대 변경 금지)

### 1. **수정 모드 분류 (= 2가지, 옛 그대로)**
- **모드 A**: `↻ 설정만 수정` (= in-place)
  - 포지션 유지 + 단계 유지 + TP/SL 만 즉시 갱신
  - 거래소 호출 X (= 사장님 자본 영향 X)
  - 시작가 무시 (= 옛 가격 보존)
- **모드 B**: `🛑 종료 후 새로 시작` (= restart)
  - 미체결 주문 취소 + 1단계부터 새 strategy 시작
  - 시작가 자동 = 현재가 적용 (옛 가격 미사용)
  - 트리거가/평단/청산가 모두 = 현재가 기준 재계산

### 2. **단계별 진입가 = 사장님 시각 우선**
- 사장님 = 수정 모드 진입 시 = **모든 단계의 진입가 = 즉시 보임**
- 진입한 단계 (= 1~N) = 실제 체결 평단 = 변경 X
- 미진입 단계 (= N+1~M) = 시작가 + 트리거% 누적 = 사장님 즉시 가시성

### 3. **SL 정책 = 투자금 × 80% 강제 청산 (= 옛 그대로)**
- SL 한도 = `total_capital × SL%` (default 80%)
- 모든 단계 진입 완료 후 = SL 발동 가능
- 레버리지 무관 = 사장님 사상 PR #57 (정확)

### 4. **사장님 자본 영향 = 모든 변경 = 검증 + 미리보기**
- 자본/트리%/단계 변경 = 즉시 = 「미리보기」 = 모든 단계 진입가 + SL 한도 표시
- 사장님 결정 전 = 모든 영향 = 시각 확인 가능

## 📐 모드 A 「↻ 설정만 수정」 — 정확 로직

### 사장님 의도:
> 포지션 + 진입한 단계 = 그대로. TP/SL 임계 % 만 = 즉시 변경.

### 변경 가능 (= 안전):
- ✅ TP1~10 % (= template tp1_percent ~ tp10_percent)
- ✅ SL % (= template sl_percent)
- ✅ stages_config 의 미진입 단계 자본/트리%/증거금 (= 사장님 의도 시)
- ✅ 마지막 단계 trigger_percent override (= 단계 추가/줄임 시)

### 변경 거부 (= 위험 차단):
- ❌ side (LONG/SHORT)
- ❌ leverage (= 거래소 호출 필요)
- ❌ symbol (= 다른 자산 = 의미 X)
- ❌ 이미 진입한 단계 의 자본/트리%/증거금

### 미진입 단계 진입가 재계산 로직 (= 사장님 명시 의도):
```
사장님 strategy: 1~4단계 진입 완료, 5/6단계 미진입.
사장님이 = 5/6단계 = 자본 또는 트리% 변경.

신 진입가 계산:
  - 평단 (= 1~4단계 가중평균) = 보존 (= 사장님 실 체결 데이터)
  - 5단계 trigger_price = 평단 × (1 + sign × 5단계 trigger_%/100)
  - 6단계 trigger_price = 5단계 trigger_price × (1 + sign × 6단계 trigger_%/100)
  - = 누적 % (= 사장님 사상 PR #60 「빈 단계 자동 압축 + trigger 누적」 그대로)

UI 표시 (= 사장님 즉시 가시성):
  - 5단계: 진입가 ??? + 트리거% (사장님 입력)
  - 6단계: 진입가 ??? + 트리거% (사장님 입력)
  - SL 한도: 총 자본 (= 1~6단계 합) × SL% = -??? USDT
```

### 사장님 헷갈림 해결 (= 즉시 가시성):
- 「미리보기」 모달 = 모든 단계 (= 진입 + 미진입) 진입가 + 누적 SL 한도 표시
- 변경 즉시 = 미진입 단계 진입가 재계산 + UI 갱신
- 사장님 결정 전 = "변경 후 SL 한도 = -X USDT" 명확 표시

## 📐 모드 B 「🛑 종료 후 새로 시작」 — 정확 로직

### 사장님 의도:
> 미체결 주문 모두 취소 + 1단계부터 = **현재가** 기준 새 strategy.

### 자동 동작:
- 모든 미체결 주문 = 취소
- 1단계 = `current_price` 즉시 진입 (= 옛 시작가 미사용)
- 2~N단계 = 1단계 평단 + 누적 트리거% (= 사장님 입력 그대로)
- TP/SL = 사장님 입력 그대로 + 새 평단 기준

### 사장님 입력 보존:
- ✅ symbol, side, leverage
- ✅ 자본 (= 단계별)
- ✅ 트리거% (= 단계별)
- ✅ TP/SL %
- ❌ 시작가 (= 옛 미사용, 현재가 자동)

## 📊 시나리오 매트릭스 (= 8 시나리오)

### S1: WAITING (= 1단계 미진입) + 모드 A
- 평단 없음 → 1단계 시작가 = 사장님 입력 그대로
- 모든 단계 진입가 = 사장님 입력 시작가 + 누적 트리%
- ✅ 안전

### S2: WAITING + 모드 B
- 1단계 시작가 = 현재가 자동
- 2~N단계 = 1단계 + 누적 트리%
- ✅ 안전 (= 옛 입력 잃음 = 사장님 의도 시 OK)

### S3: ENTERED (= 1~N단계 진입) + 모드 A
- 진입 단계 (= 1~N) = 실 체결 평단 보존 (= 변경 X)
- 미진입 단계 (= N+1~M) = 평단 + 누적 트리% (= 사장님 신 입력)
- ✅ 안전 (= 사장님 의도 + 명확 표시 필요)

### S4: ENTERED + 모드 B (= 강제 종료 + 재시작)
- 모든 미체결 주문 취소
- 진입한 포지션 = 시장가 청산 (= STOPPING)
- 1단계 = 현재가 기준 신 strategy 생성
- ⚠️ 사장님 자본 영향 (= 청산 손익 발생) → 사장님 명시 승인 필요

### S5: TP1 발동 후 + 모드 A
- 진입한 단계 + TP1 청산 25% = 그대로
- 잔여 75% qty = 변경 X
- 미진입 단계 = 평단 + 누적 트리% (= 사장님 신 입력)
- ✅ 안전 (= 사장님 의도 시)

### S6: Crisis 진입 + 모드 A
- Crisis 진행 중 (= max_loss 도달, TP 임계 -5% 자동)
- 사장님 = TP/SL % 수정 시 → Crisis 모드 시 = override 무시 (= 옛 정책)
- 정상 모드 복귀 시 = 사장님 입력 적용
- ⚠️ Crisis 중 = 사장님 입력 의미 X (= 시스템 안전 우선)

### S7: 모든 단계 진입 완료 + 모드 A
- 미진입 단계 X → 자본/트리% 변경 영향 = TP/SL 한도만
- SL 한도 = (변경된 자본 합) × SL% = 즉시 갱신
- ✅ 안전 + 사장님 자본 보호 정확

### S8: STOPPING 진입 (= 청산 진행 중) + 수정 시도
- ❌ 거부 (= TERMINAL_STATUSES에 STOPPING 포함, 옛 그대로)
- 사장님 안내: "이미 종료 진행 중 — 종료 후 새 전략 시작"

## 🛡 SL 정책 검증 (= 사장님 자본 보호)

### 사장님 사상 (PR #57 영구):
```
SL 한도 = total_capital × SL% (default 80%)
= 투자금 (사장님 자본) × 80% = 강제 청산 임계
= 레버리지 무관 (사장님 사상 명시)
```

### 모드 A 변경 후 SL 재계산:
```
변경 전:
  total_capital = 4,200 (= 6단계 자본 합)
  SL% = 80%
  SL 한도 = -3,360 USDT

변경 후 (= 사장님 = 5단계 자본 1,000 → 2,000 변경):
  total_capital = 5,200 (= 신 6단계 합)
  SL% = 80%
  SL 한도 = -4,160 USDT (= 사장님 자본 보호 강화)
```

### 시각 표시 (= UI 강화 필요):
- 「미리보기」 = 변경 전 vs 변경 후 SL 한도 = 명확 비교
- 사장님 결정 전 = 자본 보호 영향 = 즉시 인지

### Phase 2 — UI 강화 (= 사장님 즉시 가시성)

#### 2-1: 「수정 모드」 모달 = 단계별 진입가 사전 표시
- 모달 진입 시 = **현재 단계별 진입가 + 평단 + SL 한도** 즉시 표시
- 사장님이 = 자본/트리% 변경 시 = **실시간 재계산** + UI 갱신
- 결정 버튼 위 = "변경 후 SL 한도: -X USDT (변경 전: -Y USDT)" 명확

#### 2-2: 진입 단계 vs 미진입 단계 시각 구분
- 진입한 단계 = 회색 배경 + 「✅ 진입 완료」 배지 (= 자본/트리% 수정 거부)
- 미진입 단계 = 흰색 배경 + 입력 가능
- 사장님이 = 어디까지 진입했는지 = 즉시 시각

#### 2-3: 「미리보기 v2」 = 모든 단계 표 형식
```
┌────────┬──────────┬───────────┬─────────┬──────────┐
│ 단계   │ 자본     │ 트리거%   │ 진입가  │ 상태     │
├────────┼──────────┼───────────┼─────────┼──────────┤
│ 1      │ 300      │ -         │ 63,390  │ ✅ 진입  │
│ 2      │ 300      │ +10%      │ 69,729  │ ✅ 진입  │
│ 3      │ 600      │ +10%      │ 76,702  │ ✅ 진입  │
│ 4      │ 500      │ +10%      │ 84,372  │ ✅ 진입  │
│ 5      │ 1,000    │ +10%      │ 92,809  │ 🔲 대기  │
│ 6      │ 1,500    │ +10%      │ 102,090 │ 🔲 대기  │
└────────┴──────────┴───────────┴─────────┴──────────┘
총 자본: 4,200 USDT | SL 한도: -3,360 USDT (80%) | 평단: 73,X
```

### Phase 3 — Backend 검증 강화

#### 3-1: `update_strategy_settings_in_place()` 강화
- 진입한 단계 자본/트리% 변경 = **명시 거부** (= 옛 = 무조건 허용 silent bug 위험)
- 신 SL 한도 = 사장님 자본 < 신 SL 한도 시 = warning
- audit log = 모든 변경 = RiskEvent 기록

#### 3-2: 「미리보기 endpoint」 신규
- POST `/strategies/{id}/preview-edit` = 신 payload 가상 적용 + 시뮬레이션
- 응답 = 단계별 진입가 + SL 한도 + 신 평단 + 영향 분석
- UI = 사장님 결정 전 = 즉시 호출

### Phase 4 — 시나리오 자동 검증 (= 헌법 검증)

- 8 시나리오 (S1~S8) = 자동 pytest
