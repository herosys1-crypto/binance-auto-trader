# 어드민 대시보드 v2 기획서 — 운영자 친화 재설계

작성일: 2026-04-25
대상: Binance Auto Trader 운영자 (이규수)
배경: 현재 v1 (개발자 대시보드) 가 기술 용어 위주라 직관성 부족. 운영자 1명이 매일 보면서 안심하고 빠르게 판단할 수 있는 화면으로 재설계 필요.

---

## 1. 현재 v1 의 문제점

| 항목 | 문제 | 운영자 입장 |
|---|---|---|
| 상태 표시 | `STAGE1_OPEN_PENDING`, `STOPPING`, `CLOSED_BY_SL` 같은 영문 코드 | 무슨 뜻인지 매번 해석 필요 |
| 액션 부족 | "중지" 버튼만 존재. 신규 전략 시작은 Swagger UI 가야 함 | UI 와 외부 도구 두 군데 오가야 |
| 단계 시각화 없음 | 단계 컬럼에 숫자 1만 보임. 다음 단계까지 얼마나 남았는지 모름 | 추가 진입 임박했는지 한눈에 안 보임 |
| 위험 신호 약함 | 손절 임박, 청산 임박, 일일 손실 한도 도달 등 경고가 없음 | 큰 일이 터져야 알림으로 발견 |
| 데이터 깊이 얕음 | 테이블에 숫자만. 시간 추이/차트 없음 | "어제 대비 어떤지" 모름 |
| 전략 템플릿 관리 | 표시는 되지만 생성/수정/복제는 Swagger 가야 함 | 같은 문제 반복 |
| 시스템 상태 단편적 | "API Health ✅" 만 있음. scheduler, user-stream, redis, db, telegram 상태 안 보임 | 어디가 문제인지 알기 힘듦 |

## 2. 핵심 설계 원칙

> **"운영자가 매일 아침 1분만 보면 모든 게 정상인지 / 뭔가 해야 할 일이 있는지 즉시 알 수 있어야 한다."**

1. **한국어 우선** — 모든 상태/라벨은 한국어. 영문 코드는 작은 글씨로 보조 표시
2. **신호등 시스템** — 🟢 정상 / 🟡 주의 / 🔴 위험 색상으로 한눈에 판단
3. **액션 중심** — 운영자가 자주 하는 일을 화면 상단에 큼직한 버튼으로 배치
4. **계층적 정보** — 요약 → 카드 → 상세. 깊이 들어갈수록 자세한 정보
5. **시간 축 인지** — 단순 현재 상태가 아니라 "변화" 와 "추세" 를 시각화

## 4. 상태명 한국어 매핑 표

| 영문 코드 | 한국어 표시 | 신호등 |
|---|---|---|
| WAITING | 대기 중 | ⚪ |
| STAGE1_OPEN_PENDING | 1단계 주문 발송 중 | 🟡 |
| STAGE1_OPEN | 1단계 체결 대기 | 🟡 |
| STAGE_FILLED, ACTIVE | 진행 중 | 🟢 |
| TP1_FILLED / TP2_FILLED / TP3_FILLED | n차 익절 체결 | 🟢 |
| CLOSED | 정상 종료 | ⚪ |
| CLOSED_BY_SL | 손절로 종료 | 🔴 |
| STOPPING | 종료 중 | 🟡 |
| STOPPED | 종료됨 (수동) | ⚪ |
| LIQUIDATION_IMMINENT | ⚠️ 청산 임박 | 🔴 |
| KILL_SWITCH_TRIGGERED | 🚨 긴급 정지 | 🔴 |

## 5. 운영자 일상 워크플로우

### 5-1. 매일 아침 (1분 루틴)
1. http://localhost:8000/admin-ui 접속 → 자동 로그인 유지
2. 화면 상단 4개 카드 — 시스템 상태 / 활성 전략 / 오늘 손익 / 일일 손실 사용률 한 눈에
3. **🔴 신호 0개** 면 끝. 다른 일 진행
4. **🟡 신호 있으면** 해당 카드 클릭 → 상세 확인

### 5-2. 새 전략 시작 (3분 작업)
1. [➕ 새 전략 시작] 버튼 클릭
2. 거래소 계정 → 심볼 → 방향 → 템플릿 → 시작가 순서로 선택
3. **미리보기 자동 계산** — 단계별 가격/금액 표시
4. [🚀 시작] → 1단계 LIMIT 주문 즉시 발송 → 메인 화면으로 자동 이동
5. Telegram 알림 도착 확인

### 5-3. 위험 신호 발생 시
1. 메인 화면 상단에 **🔴 빨간 띠 알림** 표시
2. 알림 클릭 → 해당 전략 상세 화면 자동 이동
3. 권장 액션 표시 (예: "마지막 단계 손절 라인까지 -2% 도달. 시장 동향 확인 후 [⏸️ 일시 정지] 또는 [🛑 즉시 종료] 권장")

### 5-4. 일일 마감 (저녁 1분)
1. **📊 통계 보기** 클릭
2. 오늘 거래 / 손익 / 활성 전략 한 줄 요약
3. 다음 날 운영 계획 점검

## 6. 구현 우선순위 (Phase 별)

### Phase C-1 — 한국어화 + 신호등 (반나절)
- 모든 상태 코드 → 한국어 + 색상 매핑 (위 4번 표 적용)
- 메인 화면 카드 4개 의미 부여 (시스템/활성/손익/한도)
- 빨간/노랑/초록 색상 일관 적용

### Phase C-2 — 신규 전략 시작 모달 (1일)
- 위 3-2 의 4단계 입력 폼
- 템플릿 선택 시 미리보기 자동 계산 (`/strategies/calculate` 활용)
- 입력 완료 → `POST /strategies` + `POST /strategies/{id}/start` 연속 호출

### Phase C-3 — 전략 상세 화면 (1일)
- 단계 진행 시각화 (체크리스트 / 진행 바)
- TP/SL 라인 표시
- 활동 타임라인 (events 활용)
- 일시정지 / 즉시종료 액션 분리

### Phase C-4 — 시스템 상태 사이드바 (반나절)
- `/admin/system-status` 신규 엔드포인트 (DB ping + Redis ping + scheduler heartbeat + telegram ping)
- 신호등 8개 표시
- 30초 간격 자동 갱신

### Phase C-5 — 통계 / 차트 (1일)
- 오늘/이번주/이번달 손익 라인 차트
- 전략별 누적 PnL
- 단계별 평균 체결 시간

### Phase C-6 — 위험 알림 띠 (반나절)
- 메인 화면 상단에 위험 신호 자동 표시
- 우선순위: 청산 임박 > 손절 근접 > 일일 한도 70% 도달

## 7. 기술 선택 권고

| 항목 | v1 (현재) | v2 (권장) | 비고 |
|---|---|---|---|
| 프론트 프레임워크 | Vanilla JS + Tailwind CDN | **Next.js + TailwindCSS + shadcn/ui** | 컴포넌트 재사용 + 타입 안전 |
| 차트 | 없음 | **Recharts 또는 lightweight-charts** | TradingView-style 캔들/라인 |
| 상태 관리 | localStorage 토큰 | **Zustand 또는 TanStack Query** | API 캐싱 + 자동 갱신 |
| 실시간 | 5초 폴링 | **WebSocket (FastAPI WebSocket)** | 즉시 반영 |
| 백엔드 추가 작업 | 없음 | `/admin/system-status` + `/strategies/{id}/stage-plans` + WebSocket 채널 | API 확장 |

**MVP 단계는 v1 의 vanilla JS 그대로 유지하면서 Phase C-1 ~ C-3 만 적용** 해도 운영자 만족도 70% 도달 예상. C-4~C-6 은 점진 추가.

## 8. 작업 일정 제안 (총 5~7일)

```
Day 1 — 오전: Phase C-1 (한국어화 + 신호등)
Day 1 — 오후: Phase C-4 (시스템 상태)
Day 2 ~ 3   : Phase C-2 (신규 전략 모달)
Day 4 ~ 5   : Phase C-3 (전략 상세)
Day 6 ~ 7   : Phase C-5, C-6 (통계 + 위험 알림)
```

각 Phase 완료마다 운영자 1차 검수 → 다음 Phase 진행.
